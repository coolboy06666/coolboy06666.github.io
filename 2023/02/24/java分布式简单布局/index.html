java分布式简单布局

前言

现在公司微服务项目越来越多，每一次都去手动部署开发环境都很麻烦，所有的服务放在一台服务器上也很吃内存，所以我们搞一个简单的shell脚本来代替这部分重复的工作。

思路

基本思路很简单，仿照Jenkins拉取git上的代码，然后进行编译，运行等。
为什么不直接用Jenkins呢，原因很简单，感觉开发环境搞个Jenkins太鸡肋，很多功能用不到，还占资源。

搭建部署环境

1.服务器配置
开发环境共两台服务器
文中用82和83代替。
2.环境清单

  • Jdk环境
  • Git环境
  • Maven环境

秉着能省资源就省资源的原则，只在82上安装完整环境，83只配置最基础的jdk环境。
3.服务器互通(双机信任)
在后续的脚本中，肯定需要使用SCP进行服务器间的文件复制及其他命令，所以为了省事，免去各种操作时的输入密码环节，首先建立双机信任。

1.输入ssh-keygen -t rsa之后全部默认回车，这样就会在/root/.ssh下生成密钥文件

因为已经生成，所以此处不选择覆盖，如未生成，一直回车即可。

2.查看~/.ssh生成密钥的文件
