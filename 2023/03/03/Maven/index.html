一.maven是什么?

maven是apache基金会的开源项目，基于java语言开发，源于犹太语，意思知识的累计。

主要是以project object manager（POM）思想管理项目的构建、报告、文档。

二.maven有什么用?

1.构建项目

加入单元测试模块,将java文件与配置文件（.xml、 .properties、 .yml ）进行分离.

2.下载jar包

通过GAV坐标自动下载所需jar包,不需要再去官网苦苦找寻jar包下载地址.

所有常用jar包都可以从https://mvnrepository.com/搜索下载。

3.依赖管理

对于重复多次开发依赖jar包较多的工程,可以建立父工程,多个项目继承父工程即可.

可以直接依赖其他的工程，使用其他工程已经开发完毕的功能。

4.分模块开发

spring cloud ==> dubbo

可以将同一个工程中的不同的模块以模块工程的方式进行区分.

5.项目打包

可以快速将项目打包（jar、war、pom）

三.为什么需要使用maven?

maven提供了对项目的全方位管理,可以有效的提高开发的效率,节约磁盘的空间,最重要的是更加适合分布式项目开发.

四.maven的安装

1.下载
进入官网下载最新版本的maven: http://maven.apache.org/download.cgi#
2.解压
将下载好的zip压缩包解压到不包含中文的路径下.

bin: 主程序

boot: maven独立的类加载器

conf: settings.xml maven的全局配置

lib: jar包
3.配置

配置环境变量

MAVEN_HOME(安装根目录)

Path(path配置bin目录)

4.检查

在管理员权限的cmd界面mvn -v 查看maven安装是否成功.

五.maven仓库

仓库就用来管理maven项目的jar包的，maven的仓库分为：

maven中央仓库

国内镜像服务器

公司局域网私服

本地仓库

六.maven配置

配置文件

1.修改本地仓库位置

maven默认的本地仓库地址是在C盘的用户-当前登录用户文件夹下的.m2里， 如：C:\Users\Administrator.m2

2.配置镜像服务器，使用阿里云镜像服务器
修改settings.xml文件在mirrors中加入mirror（镜像服务器）内容.

tips：修改好全局配置文件之后，将maven安装目录下的D:\apache-maven-3.0.5\conf的settings.xml文件拷贝到新仓库根目录，仓库中的settings.xml文件和与maven安装路径下的内容最好保持一致
